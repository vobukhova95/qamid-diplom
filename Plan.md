# 1. Общее описание приложения
Мобильное приложение «ВХосписе» — это информационное приложение, предназначенное для работы с новостями хосписа.


Приложение предоставляет пользователю:

- просмотр новостей
- фильтрацию и сортировку новостей
- административное управление новостями (создание, редактирование, удаление, изменение статуса)
- просмотр тематических цитат
- доступ к информации о приложении (версия, правовые документы)

Приложение содержит пользовательскую и административную функциональность (панель управления)

# 3. Задачи тестирования

- Проверить корректность основных функций приложения: авторизацию пользователей, работу с новостями и цитатами, а также доступ к информации о приложении.
- Убедиться в правильной работе создания, редактирования и удаления новостей, включая обработку пустых или некорректных данных.
- Проверить корректность сортировки и фильтрации контента для удобного поиска и навигации.
- Обеспечить стабильность и адаптивность интерфейса на различных устройствах и разрешениях экрана, включая поддержку светлой и тёмной темы.
- Проверить работу приложения в разных сетевых условиях, при внешних событиях и восстановлении соединения.
- Оценить производительность: скорость отклика и загрузки страниц для комфортного пользовательского опыта.


# 4. Виды тестирования

- **Функциональное тестирование** – проверка работы всех функций приложения, включая авторизацию, работу с новостями и цитатами, сортировку и фильтрацию.
- **Тестирование пользовательского интерфейса (UI)** – проверка корректного отображения элементов на разных устройствах, разрешениях экрана и темах (светлая/тёмная).
- **Тестирование производительности** – оценка скорости отклика приложения, времени загрузки главной страницы и списка новостей.
- **Тестирование адаптивности и совместимости** – проверка работы приложения на разных устройствах, версиях ОС, разрешениях экранов и DPI.
- **Тестирование работы в различных сетевых условиях** – проверка поведения приложения при слабом или отсутствии интернета, использовании VPN и восстановлении соединения.
- **Тестирование обработки внешних событий** – проверка корректной работы приложения при входящих звонках, SMS, сворачивании и блокировке устройства.

# 10. Техники тест-дизайна с обоснованием

Для эффективного тестирования приложения «ВХосписе» будут использованы следующие техники тест-дизайна:

## 1. Эквивалентное разбиение
- **Описание:** Разделение входных данных и состояний приложения на группы (классы эквивалентности), в пределах которых поведение системы считается одинаковым.  
- **Применение:** Для проверки полей авторизации (логин, пароль) можно выделить группы: корректные данные, пустые поля, некорректные форматы. Для фильтров новостей – диапазоны дат, категории (существующие и несуществующие).  
- **Обоснование:** Позволяет уменьшить количество тестов, сохранив покрытие всех возможных сценариев.

## 2. Анализ граничных значений
- **Описание:** Тестирование на граничных значениях входных данных, где часто возникают ошибки.  
- **Применение:** Для дат фильтрации новостей – проверка крайних дат диапазона. Для текста новости – проверка минимальной и максимальной длины заголовка или описания.  
- **Обоснование:** Позволяет выявить ошибки обработки предельных значений, которые часто не обрабатываются корректно.

## 3. Сценарное тестирование
- **Описание:** Создание тестов на основе сценариев использования приложения пользователями.  
- **Применение:** Проверка полного сценария: авторизация → просмотр новостей → фильтрация и сортировка → просмотр цитаты → выход из приложения.  
- **Обоснование:** Позволяет проверить бизнес-логику приложения и убедиться, что пользовательский путь работает целиком.

## 4. Комбинационное тестирование
- **Описание:** Проверка комбинаций входных данных, когда все возможные комбинации непрактично тестировать полностью.  
- **Применение:** Для фильтров: сочетания категории и диапазона дат. Для настройки интерфейса – светлая/тёмная тема × разные разрешения экранов × разные устройства.  
- **Обоснование:** Позволяет покрыть наиболее вероятные сочетания условий без тестирования всех возможных вариантов.

## 5. Исследовательское тестирование
- **Описание:** Тестирование без строгого сценария, с изучением приложения и попыткой найти нестандартные ошибки.  
- **Применение:** Для новых функций и сложных интерфейсных элементов, где заранее трудно предсказать все сценарии использования.  
- **Обоснование:** Позволяет обнаружить ошибки, которые не покрываются формальными тест-кейсами.
